Installation

The program will be supplied by Tony. It requires the Java Run Time Environment (JRE) to be installed on your machine

First usage

The first time you start the program use the MPCORB button to select your local copy of MPCORB.DAT.


Options

  • Refresh: Refresh all data.
  • (Observatory selection): Select G40 or W88.
  • Ephemeris: The MPC Ephemeris page will be displayed for those the selected non-NEOCP objects.
  • NEOCPs: The MPC Ephemeris page will be displayed for NEOCP/PCCP objects.
  • +24hrs: Show data for the following night.
  • ClearSel.: Turn off all selections.
  • Where?: List visibility times of the selected objects at either observatory.
  • Filter: Press this button to refresh the list after changing filter settings.
  • MPCORB: Use this button to select the local copy of MPCORB.

Filter

  • Twilight: Select the twilight to apply when calculatibing earliest visibility.
  • Maximum magnitude download: Objects in the various downloaded lists with current magnitudes fainter than this are ignored.
  • Maximum magnitude display: Objects with a current magnitude fainter than this are not displayed.
  • Minimum rise/set altitude: The rise and set times are established based on a horizon set at this altitude. An object must appear above this horizon to appear in the list.
  • Minimum galactic latitude: Objects with a Galactic Latitude less than this are excluded from the list.
  • Show asteroids with uncertainty >=: Excludes objects with a lower uncertainty. Does not apply to NEOCPs.

Mission Time Filter

You can set Mission Times From -To to constrain the display of objects. Be careful to ensure the times are somewhere between sunset and sunrise.

  • All of mission period: Objects are only listed if they are visible throught the From-To times.
  • Not within 30m of Meridian: Objects must be visible and not positioned within 30 minutes of meridian crossing.

Note that you can also use the From time to control the start time of the MPC Ephemeris listing.

I use robotic telescopes to capture images of Near Earth Asteroids and newly discovered comets. I measure their positions and magnitudes, and report this information to the Minor Planet Centre (MPC).

I have devloped some small software tools to help me. Some of them are also used by my friends in the Slooh.com A-Team (a group of like-minded asteroid and comet chasers using the Slooh.com robotic telescopes.

I am placing these tools on Github as a convenience to the A-Team members and in case they may be of any use to anyone else.

ADES Comet

This takes separate positional (astrometric) and magnitude (photometric) observations and combines them into single ADES observations containing both the astrometric and photometric information, ready to send to the MPC.

ADES COMET

ADES to Obs80

This takes ADES data, as output from Astrometrica and converts it to the old Obs80 format

ADES to Obs80

NEOCP Data extract

This downloads data from the MPC detailed records of NEOCP objects and extracts the observations in obs80 format.
